Discussion

add subtree under a child node of this tree, and add the map info in subtree root into the root of this tree (assuming it’s this node) @note call this method only on the root node, and make sure that node is an offsprint under this node

if the subtree has a conflicting ID with the original tree, the ID will refer to the one in the new subtree

newDepthFirstOrderDataArray

create an array with the content node data in this tree in depth first order (natural markup string order)

- (NSArray *)newDepthFirstOrderDataArray

Return Value

an array of dictionaries, each dictionary is the data of a content node @example a tree based on “a BC D ” will give an array of [a, B, C, D] (styles included) @note only content nodes are exported. Container nodes are not. This makes output lighterweight. @note only data (dictionary) is stored in the array because you don’t need tree information (children, parent) when it’s flattened @note new means that you own it (it’s not autoreleased, and you have to release it upon finishing, if it’s not ARC)

Discussion

create an array with the content node data in this tree in depth first order (natural markup string order)

resetChildToText:

Reset the text under this node

- (void)resetChildToText:(NSString *)text

Discussion

Reset the text under this node

if there’s any subtree under this node, it will be removed. After this call there will be only one text child note, plus the lambda nodes @example in this tree “ABCXXX, if we call resetChildToText:@"123” to id=1 node, it will become 123; and in “ABCDEF”, if we can the same method, it will give us “123”
